
# Security Policy

## Private Information

Never commit:

* WordPress admin passwords
* Supabase service role keys
* API keys
* cPanel credentials
* FTP credentials
* Database passwords
* Private customer data
* Driver personal documents
* Payment details

## Recommended Security Rules

* Use strong WordPress admin passwords.
* Enable SSL on daynightae.com.
* Keep WordPress themes and plugins updated.
* Avoid nulled or cracked WordPress plugins.
* Store secrets in hosting control panel or GitHub Secrets.
* Do not expose Supabase service role keys in frontend code.

## Reporting

Security issues should be handled privately by the project owner.
